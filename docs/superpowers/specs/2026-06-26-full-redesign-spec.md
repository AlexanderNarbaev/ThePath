# Design Spec: Полный редизайн «Спирали Сознания»

**Дата:** 2026-06-26  
**Статус:** draft  
**Версия:** 1.0  

---

## 1. Контекст и цели

### 1.1 Исходная проблема

Сайт «Спирали Сознания» (57 страниц, Astro 5 + Preact + Tailwind 4) получил критику от реального посетителя (Артем Воронин, инженер-системщик):

| Критика | Суть |
|---------|------|
| «Слишком много маркеров ИИ-генерации» | Типографика, иконки, навигация выглядят шаблонно |
| «Чувство будто obsidian заметки читаешь» | Контент как личная база знаний, не публикация |
| «Втянуть народ слишком сложно» | Нет hook'а, порог входа завышен |
| «Надо бы верстку формы подправить» | Форма обратной связи сломана |
| «Формировать костяк исключительностью» | Нет уникального характера, визуального почерка |

### 1.2 Результаты трёхстороннего аудита

**Контент-аудит:** 3.6/10 — отсутствие hook'а, эффект «Obsidian notes», религиозная лексика при антирелигиозном позиционировании, разрыв практика/теория, нет social proof.

**Код-аудит:** 6.0/10 — контактная форма сломана (`PUBLIC_WEB3FORMS_KEY` не доходит до клиента), 11 компонентов на `client:load`, дубликация CSS/HTML, нет `:focus-visible`, `<dt>/<dd>` без `<dl>`, 7 `as any`.

**Дизайн-аудит:** 5.0/10 — эмодзи вместо иконок, отсутствие визуальной идентичности, навигация без keyboard support, тёмная/светлая тема с грубым переходом.

### 1.3 Цели редизайна

1. **Удержание с первой секунды** — hook, который заставляет скроллить
2. **Уникальный визуальный почерк** — не AI-генерированный, не шаблонный
3. **Честность как фундамент** — никакого маркетинга, никакой «сектантской» лексики
4. **Практика прежде теории** — перевёрнутая пирамида контента
5. **Техническое качество** — a11y, производительность, отсутствие багов

### 1.4 Non-goals

- Не меняем ядро Канона (Модуль 0) — только лексику и подачу
- Не добавляем бэкенд/базу данных
- Не меняем хостинг (GitHub Pages)
- Не создаём новые модули
- Не добавляем авторизацию/аккаунты

## 2. Визуальный язык: «Космический брутализм»

### 2.1 Базовые принципы

Утверждённый гибрид трёх стилей:

| Источник | Что берём |
|----------|-----------|
| **Космическое тепло** (основа) | Тёмный фон `#0a0a14`, градиенты, золотой акцент, воздушная типографика |
| **Бруталистский минимализм** | Моноширинный шрифт для мета-комментариев, прямой язык, стрелки (▸ →) вместо декора, никаких эмодзи |
| **Академическое издание** | Цитаты и сноски, прозрачная структура, факты вместо обещаний |

### 2.2 Четыре правила

1. **Тёмный космос — фон.** Глубокий тёмный `#0a0a14`, градиенты, золотой `#e6c17b` как сигнальный цвет. Светлая тема остаётся для длинного чтения (срабатывает по системной настройке).

2. **Два типографических голоса.**
   - *Основной:* системный sans-serif (Inter, -apple-system) — тёплый, читаемый, для контента
   - *Честный:* моноширинный (JetBrains Mono) — для мета-комментариев, фактов, предупреждений, «#НЕ_РЕЛИГИЯ»
   - Никаких эмодзи. Только текстовые символы (▸, →, ✓, ✕).

3. **Золото — сигнал, не декор.** Используется только для смысловых акцентов: ключевые фразы, разделители, кнопки действий. Не для «красивости».

4. **Никакого маркетинга.** Никаких «присоединяйтесь», «уникальная возможность», «тысячи людей уже...». Только: «Вот система. Вот практики. Вот почему это работает. Решать тебе.»

## 3. Архитектура лонгрида (главная страница)

### 3.1 Нарративная арка (Гибрид)

```
┌─────────────────────────────────────────────┐
│ §0  HERO         │ Боль + прозрачность + CTA │ ← 5 секунд
├─────────────────────────────────────────────┤
│ §1  ПРАКТИКА     │ 3 минуты. Прямо сейчас.   │ ← немедленная ценность
├─────────────────────────────────────────────┤
│ §2  ПРОЗРАЧНОСТЬ │ Что это — и чем не является│ ← снятие скепсиса
├─────────────────────────────────────────────┤
│ §3  НАУКА+ЭТИКА  │ Почему это работает        │ ← доверие
├─────────────────────────────────────────────┤
│ §4  КАРТА        │ Что дальше (добровольно)   │ ← развилка
└─────────────────────────────────────────────┘
```

### 3.2 Секция §0: Hero

**Задача:** за 5 секунд ответить на вопрос «почему мне это нужно?» и дать одно действие.

**Структура:**
- Мета-лейбл (mono): `# ОТКРЫТАЯ СИСТЕМА · НЕ РЕЛИГИЯ`
- Заголовок (sans, light): `Ваше внимание — единственное, что вам по-настоящему принадлежит.` (золотая фраза-якорь)
- Подзаголовок: `3 практики. Никакой магии. Никакой веры. Только вы и ваши решения.`
- Одна кнопка: `→ 3 минуты сегодня` (золотая, ведёт к §1)
- Ссылка-намёк: `↓ что это` (ведёт к §2)

**Тон:** не продающий. Не «присоединяйтесь». Не «узнайте». Просто: «Вот что есть. Хочешь — попробуй.»

### 3.3 Секция §1: Практики

**Задача:** немедленная actionable ценность. Посетитель пробует до того, как понимает.

**Структура:**
- Мета-лейбл: `§1 · ПРАКТИКА`
- Заголовок: `Три минуты. Прямо сейчас.`
- Три карточки: Утро / Перед экраном / Вечер (каждая: иконка-символ, текст практики, тайминг)
- Интерактивный мини-чеклист «Сегодня» (3 галочки, localStorage)
- Кнопка: `Отметить сегодня ✓`

**Важно:** чек-лист — якорь возврата. Посетитель может вернуться завтра и отметить снова.

### 3.4 Секция §2: Прозрачность

**Задача:** снять главное возражение «это секта?» до того, как оно возникнет.

**Структура:**
- Мета-лейбл: `§2 · ПРОЗРАЧНОСТЬ`
- Заголовок: `Что это — и чем это не является`
- Сетка 3×2: НЕ / ДА
  - ✕ НЕ Религия → ✓ ДА Практическая система
  - ✕ НЕ Идеология → ✓ ДА Открытый код
  - ✕ НЕ Гуру → ✓ ДА Сообщество равных

**Тон:** прямой, без оправданий. «Нет, это не религия. Да, это инструмент. Нет, у нас нет лидера.»

### 3.5 Секция §3: Наука + Этика (дизайн уточняется при реализации)

**Задача:** показать, что за практиками стоит реальная база.

**Содержание:**
- Термодинамика и энтропия внимания
- Нейронаука привычек
- Эволюционная этика кооперации
- Три запрета как рациональный выбор
- Четыре принципа расцвета

**Формат:** раскрывающиеся блоки (details/summary) или карточки с иконками. Не стена текста.

### 3.6 Секция §4: Карта

**Задача:** дать развилку для тех, кто хочет глубже.

**Структура:**
- Заголовок: `Что дальше`
- Три пути (карточки): Быстрый старт / Наставник / Исследователь
- Каждый путь: краткое описание, время, ссылка на страницу пути
- Ссылка на все модули

## 4. Лексическая реформа (гибридный подход)

### 4.1 Таблица замен

| Было | Стало | Обоснование |
|------|-------|-------------|
| Символ веры | **Слова выбора** | «Вера» — религиозный триггер. «Выбор» — честно. |
| Да будет так | **Это мой выбор** | Ритуальная формула → личная декларация |
| Хранитель Сознания | **Практикующий** | «Хранитель» пахнет сектой. |
| обеты | **обязательства** | Светский синоним |
| посвящение | **вступление** | Без ритуальной коннотации |
| Канон | **Канон** (оставить) | Как «канон врачебной науки» — технический термин |
| прихожанин | **участник** | Нейтрально |

### 4.2 Что остаётся

- **Спираль Сознания** — название системы
- **Три запрета** — сильная этическая позиция
- **Четыре принципа** — ясная структура
- **Манифест** — жанр, не религия

### 4.3 Область применения

Замена применяется в:
- UI (навигация, заголовки, кнопки)
- i18n/ui.ts (все ключи)
- Контент модулей (frontmatter и тело)
- Манифест и Карта
- FAQ

НЕ применяется в:
- Архивных документах (`docs/Archive/`)
- Легаси-версиях сайта (`docs/ThePath/`)

## 5. Технический рефакторинг

### 5.1 Критические баги (блокируют деплой)

| # | Проблема | Файл | Исправление |
|---|----------|------|-------------|
| 1 | `PUBLIC_WEB3FORMS_KEY` не доходит до клиента | `ContactForm.tsx:23` | Передать ключ как prop из Astro-компонента |
| 2 | `<dt>/<dd>` без `<dl>` | `glossary.astro:44` | `<div>` → `<dl>` |

### 5.2 Производительность

| # | Проблема | Исправление |
|---|----------|-------------|
| 1 | 11 компонентов на `client:load` | 8 перевести на `client:visible`, 1 на `client:idle` |
| 2 | D3 (~80KB) для одного графа | Заменить на CSS Grid или `<canvas>` |
| 3 | lunr (~20KB) для 19 сущностей | `Array.filter()` вместо lunr |

**План `client:` директив:**
- `client:load` → только `ProgressBar`, `SearchModal`
- `client:visible` → `BookmarkButton`, `Reactions`, `DailyChecklist`, `EthicalCompass`, `PathTracker`, `ModuleGraph`, `VisitorCounter`, `ContactForm`
- `client:idle` → `EthicalSimulator`

### 5.3 Извлечение общих компонентов

| Новый компонент | Заменяет | Экономия |
|-----------------|----------|----------|
| `CardsGrid.astro` | `ProhibitionsCards` + `PrinciplesCards` | ~70 строк |
| `src/utils/static-paths.ts` | 5 файлов с `getStaticPaths` | ~25 строк |
| `src/hooks/useLocalStorage.ts` | 5 компонентов | ~60 строк |
| `src/constants.ts` | 13 файлов с `const base = '/ThePath'` | ~13 строк |

### 5.4 Удаление

| Что | Почему |
|-----|--------|
| `HomeLayout.astro` | 9 строк, не добавляет ценности над `BaseLayout` |
| `@preact/signals` из dependencies | Не используется нигде |
| Tailwind (опционально) | 95% стилей — ручной CSS; либо удалить Tailwind, либо мигрировать все стили на utility-классы |

### 5.5 Accessibility (WCAG 2.1 AA)

| # | Проблема | Исправление |
|---|----------|-------------|
| 1 | Нет `:focus-visible` нигде | Добавить на все интерактивные элементы |
| 2 | Nav-toggle без `aria-expanded` | Добавить `aria-expanded` + `aria-controls` |
| 3 | Nav-toggle/ThemeToggle без keyboard | Добавить `onkeydown` (Enter/Space) |
| 4 | Контраст gold на cosmic: 3.4:1 | Использовать gold только для крупного текста/бордеров (проходит для 18px+) |
| 5 | ModuleGraph без keyboard | Добавить `tabindex` + `keydown` на узлы |

## 6. Контентные изменения

### 6.1 Home page (переписывается полностью)

Текущий `[lang]/index.astro` заменяется на лонгрид. Текущий контент из `ru/index.md` / `en/index.md`:
- Секция «Что это» → распределяется по §1 и §2
- Секция «Для кого» → перемещается в §4 (Карта)
- Секция «Почему работает» → в §3 (Наука)
- Таблица модулей → уходит на страницу `/map`

### 6.2 Убираем «Obsidian notes»

| Признак | Решение |
|---------|---------|
| Change-log в теле документов | Убрать с публичных страниц. Хранить в GitHub releases или `CHANGELOG.md` |
| Нумерация версий в футере | Убрать `footer.version`. Оставить только в `<meta>` модулей |
| Markdown-таблицы как интерфейс | Заменить на карточки / сетки |
| Перекрёстные ссылки без контекста | Добавить нарративный переход или встроить краткое объяснение |
| Линейная нумерация §1..§N | Заменить на осмысленные заголовки |
| FAQ как приложение к техдоку | Вынести на отдельную страницу «Вопросы и сомнения» |

### 6.3 Новые страницы

| Страница | Назначение |
|----------|------------|
| `/questions` | FAQ, стилизованный под диалог (не раздел спеки) |
| `/start` | «Твой первый день» — лэндинг с 3 практиками для новичков |

## 9. Критические ограничения

1. **Канон (Модуль 0) — ядро неизменно.** Меняется только лексика подачи, не смысл.
2. **Манифест — минимальные правки.** Это лучший текст сайта (по оценке аналитика и пользователя).
3. **Все внутренние ссылки — с `/ThePath/` префиксом.**
4. **Никаких внешних ссылок на сайт.**
5. **CC BY-SA 4.0 — лицензия не меняется.**

## 10. Самопроверка (spec review)

- **Placeholder scan:** нет TBD, нет TODO — все секции специфицированы.
- **Internal consistency:** визуальный язык → компоненты → контент → лексика согласованы. Нарративная арка лонгрида соответствует визуальному стилю.
